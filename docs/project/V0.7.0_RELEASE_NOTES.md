# Church Cap v0.7.0 Release Notes

## Commit Summary

Church Cap v0.7.0 is a major public-preview update focused on smoother live captions, responsive contextual translation, diagnostics, and release readiness.

## Commit Description

This release improves Church Cap as a local-first live caption system for churches. Pilot feedback and repeatable Mac and Linux measurements informed changes that make captions easier to follow, reduce repeated or jumping text, improve contextual translation without paragraph-length waits, and keep diagnostic storage bounded while service-time processing remains offline.

Highlights:

- Add Faster-Whisper word timestamps, confidence-aware live-edge guarding, sealed-audio trimming, and start-to-start recognition cadence.
- Give English and translated captions stable cue identity so corrected drafts update in place instead of appending duplicate lines.
- Use one viewport-bounded, internally scrollable Live reader across English, translated languages, left-to-right, and right-to-left layouts; start Session transcript closed.
- Add Responsive Context, which translates stable English cue text without waiting for two- or four-thought paragraph batches.
- Add fair, bounded per-language translation queues with revision coalescing, final-cue durability, overload recovery, and bounded Stop draining.
- Separate Simplified Chinese and Hong Kong Traditional Chinese, with deterministic OpenCC script conversion.
- Advance anonymised service reports to schema 9 with word-alignment, cue, queue, translation-publication, first-translated-cue, shutdown, and resource measurements.
- Add Operator → Diagnostics storage accounting and allow-listed cleanup; rotate logs and bound diagnostic and benchmark data.
- Preserve sensitive-moment and transcript privacy boundaries while improving local transcript controls and exports.
- Update macOS, Windows, Linux, appliance, user, architecture, translation, testing, and legal documentation; 179 automated tests pass.
- Refresh the web dependency pins after a release audit and disable Argos's unused Stanza sentence-boundary path for short Church Cap cues.

Notes:

Church Cap remains local-first and open-source. Audio, transcription, translation, and normal service use run on the Church Cap computer and local network. Operators must still test the selected model, audio route, languages, and performance settings on their own hardware before relying on the app in a live service. AI captions and translations can contain errors and should not be treated as a verbatim or authoritative record.

## GitHub Release

### Church Cap v0.7.0

This release improves the readability, translation responsiveness, diagnostics, and release quality of Church Cap while retaining its local-first design.

### Highlights

- Smoother English and translated captions with stable cue identity and correction-in-place.
- Faster-Whisper word timestamps and guarded weak live-edge words for fewer distracting partial-word corrections.
- Responsive contextual translation sourced from stable English wording, without slow paragraph batching.
- Separate Simplified Chinese and Hong Kong Traditional Chinese choices.
- Fair bounded translation scheduling, draft coalescing, final-cue durability, and measurable first-translation delay.
- A consistent bounded scrolling Live reader for all languages, with Session transcript closed by default.
- Diagnostics storage accounting, safe cleanup, bounded benchmark data, and log rotation.
- Expanded anonymised schema 9 service reports, updated cross-platform documentation, and 179 passing automated tests.
- Refreshed web dependencies and tightened the optional Argos model-loading boundary after a security audit.

### Notes

Church Cap remains local-first and open-source. Audio, captions, translations, transcripts, diagnostics, and configuration remain on the Church Cap computer during normal service use. AI-generated captions and translations can contain mistakes, so churches should test their intended hardware and languages and provide an appropriate notice to viewers.
