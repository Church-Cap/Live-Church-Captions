# Church Cap v0.7.1 Release Notes

## Commit Summary

Church Cap v0.7.1 is a focused maintenance update that improves language settings, Service Leader onboarding, operator navigation, updater version handling, and release documentation while preserving the caption and translation behaviour introduced in v0.7.0.

## Commit Description

This release builds on the measured caption, translation, and diagnostics work in v0.7.0. It resolves four small operator-interface issues and one release-updater version-reader issue without changing transcription cadence, cue publication, translation scheduling, diagnostic schemas, or audience caption behaviour.

Highlights:

- Add a prominent **Save language settings** button directly below the maximum active translated languages control; retain the existing advanced-settings save button.
- Make Recommended translation-mode guidance use the running application's version label instead of a stale hard-coded v0.6.x reference.
- Add an on-page explanation of the Service Leader role, its intended controls, and the settings that remain operator-only, presented in the same rounded Church Cap card style as the surrounding controls.
- Replace the ambiguous Diagnostics and Updates menu marks with clearer inspect/search and download-to-tray icons that use the existing Church Cap theme.
- Update code, documentation, tests, release metadata, and the action plan to the v0.7.1 naming scheme.
- Update macOS/Linux and Windows release readers to recognise the code-owned `APP_VERSION` constant, retain legacy settings compatibility, and display tags as `v0.7.1` rather than `v.0.7.1`.
- Remove the stale duplicate `.env.example`; retain the current `env.example` and backward-compatible setup-script fallback handling.
- Retain the v0.7.0 word-timestamp cue engine, Responsive Context translation, bounded queues, Chinese script choices, viewer behaviour, and diagnostics unchanged.
- Verify 186 automated tests, Python and JavaScript compilation, shell syntax, workflow/JSON parsing, authenticated page rendering, browser presentation, Markdown links, dependency health, and the rendered action-plan document.
- Confirm no known vulnerabilities in the default requirements; document the two upstream-pinned advisories in the optional Argos dependency set and its trusted-model boundary.

Notes:

Church Cap remains local-first and open-source. Audio, transcription, translation, and normal service use run on the Church Cap computer and local network. Operators must still test the selected model, audio route, languages, and performance settings on their own hardware before relying on the app in a live service. AI captions and translations can contain errors and should not be treated as a verbatim or authoritative record.

## Release Summary

Church Cap v0.7.1 improves everyday operator clarity and updater reliability while retaining the local-first caption and translation behaviour from v0.7.0.

## Release Description

Church Cap v0.7.1

This maintenance release improves operator clarity while retaining the local-first caption and translation behaviour from v0.7.0.

### Highlights

- A visible language-settings save action beside the everyday translation controls, with the advanced save action retained.
- Version-aware Recommended translation-mode guidance.
- Clear Service Leader role guidance for new operators.
- More recognisable Diagnostics and Updates navigation icons in the existing Church Cap visual style.
- Reliable code-owned version detection and consistent `v0.7.1` labels in the cross-platform updater.
- Removal of the stale duplicate environment template, leaving one current `env.example` source of setup defaults.
- No changes to caption latency, translation scheduling, diagnostics schemas, privacy boundaries, or audience-facing behaviour.
- 186 automated tests pass, with additional runtime, syntax, link, dependency, browser, and document-render checks completed.

### Notes

Church Cap remains local-first and open-source. Audio, captions, translations, transcripts, diagnostics, and configuration remain on the Church Cap computer during normal service use. AI-generated captions and translations can contain mistakes, so churches should test their intended hardware and languages and provide an appropriate notice to viewers.

The default dependency set had no known vulnerabilities in the release audit. Optional Argos translation still inherits two documented upstream-pinned dependency advisories; Church Cap disables the unused Stanza model pipeline and operators should install only trusted translation packages. See `docs/legal/THIRD_PARTY_NOTICES.md` for the exact advisory and model-trust details.
