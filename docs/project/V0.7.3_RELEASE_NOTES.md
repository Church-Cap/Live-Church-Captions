## Release Description

# Church Cap v0.7.3

Church Cap v0.7.3 is a maintenance release focused on end-of-speech caption reliability. It is designed as the direct upgrade from the current public v0.7.0 release and includes the intervening operator and updater improvements.

## End-of-speech caption reliability

- Stops a decode that began after the recent-voice grace period from replacing the last speech-backed caption.
- Keeps enough trailing audio for a reliable endpoint decision, then trims it with a 160 ms Silero end-of-speech rule and 100 ms acoustic pad; active-speech windows remain unchanged.
- Uses Whisper's own no-speech and average-log-probability metadata to reject weak results before publication.
- Enables Faster Whisper's timestamp-aware hallucination-silence handling for word-timestamp runs.
- Keeps the existing start-to-start recognition cadence: there is no second verification decode and no new fixed caption wait.
- Applies the same publication policy to Faster Whisper and standard Whisper.
- Adds text-free diagnostics counters for guard interventions and speech-bounded input trimming; rejected wording and audio are never stored in the service report.

## Updater reliability

- Restores the direct **Operator → Updates** path from v0.7.0.
- Preserves the legacy release field required by the published v0.7.0 updater while keeping the running application version code-owned.
- Detects updater failure during the normal reconnect cycle instead of leaving an indefinite spinner.
- Offers **Download diagnostics** and **Return to Updates** when an update stops.
- Shows a distinct restart-needed state if file replacement finishes but Church Cap does not restart automatically.
- Downloads the exact release selected by Church Cap and rejects a second update while one is already active.
- Keeps update status privacy-safe: no absolute paths or raw log contents are returned by the status endpoint.

## Operator improvements included

- Adds a prominent **Save language settings** button beside the everyday translation controls while retaining the Advanced save button.
- Makes Recommended translation-mode guidance use the running Church Cap version instead of obsolete version text.
- Explains the restricted Service Leader role for new operators.
- Uses clearer Diagnostics and Updates menu icons in the existing Church Cap theme.
- Removes the stale duplicate environment template while keeping older setup checkouts supported.

## Behaviour retained

Word-timestamp cue handling, correction-in-place, Responsive Context translation, separate Simplified and Hong Kong Traditional Chinese choices, bounded translation queues, viewer presentation, privacy boundaries, and storage controls remain unchanged. Service metrics retain schema 11 and identify the refined guard as `silero_endpoint_progress_publication_guard_v3`; schema-9 and schema-10 histories remain readable.

## Upgrade note

Once v0.7.3 is published as the latest normal GitHub release, users on v0.7.0 can update directly through **Operator → Updates**. The literal `app_version: str = "0.7.3"` field is intentional and must remain aligned with `APP_VERSION` until v0.7.0 upgrade support is formally retired.

If an update stops, use **Download diagnostics** on the update panel. The confirmed diagnostics export includes a redacted tail of `logs/update.log`; it does not include captions, translations, transcripts, audio, passwords, or `.env` contents.

## Validation and dependency note

All 206 automated tests pass, alongside syntax, route/render, documentation, archive, published-v0.7.0 compatibility, temporary replacement, and action-plan render checks. The default dependency audit reports no known vulnerabilities.

Optional Argos translation still resolves SentencePiece 0.2.0 (`PYSEC-2026-1909`) and Stanza 1.1.1 (`PYSEC-2026-3075`) because Argos 1.9.6 pins them. Church Cap disables the unused Stanza model pipeline; install only trusted model packages and see `docs/legal/THIRD_PARTY_NOTICES.md` for the full boundary.

Church Cap remains local-first and open-source. Operators should test their intended audio route, model, languages, and hardware before a live service. AI-generated captions and translations can contain mistakes and should not be treated as a verbatim or authoritative record.
