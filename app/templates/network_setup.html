Local hostname, HTTPS, and Wi-Fi setup

Current app URL detected as {{ base_url }}. By default the app now prefers the Mac's Bonjour/mDNS hostname instead of an IP address. For a church install, use a stable hostname or reserved IP so the QR code does not change.

Friendly local hostname

On macOS, you can use Bonjour/mDNS so phones can open church-cap.local:

sudo scutil --set ComputerName "Church Cap"
sudo scutil --set LocalHostName "church-cap"
sudo scutil --set HostName "church-cap.local"

The first-time setup script can offer to set this for you on Apple Silicon Macs. If guest Wi-Fi blocks mDNS/Bonjour, use a router DNS entry or a reserved IP instead.

Then set this in .env:

PUBLIC_BASE_URL=http://church-cap.local:8080

HTTPS options

Local HTTPS has an important browser-security limitation: the caption box cannot silently install a trusted certificate on visitors' phones. Self-signed certificates will warn unless each device trusts the certificate authority. For no-warning HTTPS, use a real domain plus a DNS-01 certificate, or install a local CA on managed church devices.

./scripts/generate-local-cert.sh
./scripts/run-dev-https.sh

For trusted testing on your own Mac, install mkcert and run ./scripts/generate-trusted-local-cert-macos.sh. Visitors' phones will still need the mkcert root CA installed to avoid warnings.

Guest Wi-Fi rules

Recommended rule:

Guest Wi-Fi → allow access only to caption computer port 8080/8443
Guest Wi-Fi → block access to other LAN devices
Operator page → password protected

If .local does not work, the Wi-Fi may block mDNS/Bonjour. Use a reserved IP or router DNS entry instead.
